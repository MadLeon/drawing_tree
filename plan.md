使用 session skill 来管理当前工作流, 下面是 input 内容:

本session主要任务: 添加图纸关系构建功能

用例: 
1. 用户点击 build drawing tree 后, 
	- 先判断当前程序目录下是否包含"{PO}_import.json"文件, 如果没有, 则弹出一个提示框提示用户先使用导入图纸功能生成图纸文件
	- 若.json文件存在, 弹出一个对话框, 包含一个提示, 一个下拉菜单和一个确认按钮	
	- 下拉菜单的选项取值为当前程序目录下的所有.json文件名, 该文件由导入图纸功能生成
	- 用户选择一个PO号后点击确认, 进入图纸关系构建界面

2. 图纸关系构建界面包含一个上边栏, 类似导入图纸界面, 包含以下元素:
	- 一个文本提示, 显示当前选择的PO号, 左对齐
	- 按钮组, 包含以下按钮, 右对齐:
		- "保存"按钮: 点击后将当前构建的图纸关系保存到一个新的.json文件中, 文件名为"{PO}_tree.json", 该文件包含图纸之间的层级关系信息
		- "返回"按钮: 点击后停留在当前界面

3. 图纸关系构建界面分为左右两个部分, 左侧为图纸列表, 右侧为关系构建区域
	- 用户可以通过拖拽的方式将左侧图纸列表中的图纸添加到右侧关系构建区域，以更好地展示它们之间的关系
	- 每个图纸列表的图纸项都包含图纸号, 单击该图纸号会打开该图纸的PDF文件, 拖拽则可以将其进行移动到右侧的关系构建区域
	- 右侧区域应该呈现一种文件夹树的结构, 用户第一次拖拽的图纸会成为树的根节点, 后续拖拽的图纸可以被放置在根节点下或者其他图纸节点下, 以构建出图纸之间的层级关系
	- 右侧的图纸元素应该也能够被单击文本以打开对应的PDF文件, 以及被拖拽以调整它们在树结构中的位置
	- 右侧的图纸元素应该包含+/-按钮
		- 点击+号展开该节点下的子节点, 并改变该按钮为-号
		- 点击-号则收起该节点下的子节点, 并改变该按钮为+号
		- 只有包含子节点的图纸元素才显示+/-按钮, 没有子节点的图纸元素不显示该按钮
	- 当一个元素被拖拽到另一个元素上时, 下方的元素应该显示一个高亮的下边框, 以提示用户可以将其放置在该位置
	- 当用户将一个元素A放置在另一个元素B上时, 元素A应该成为元素B的子节点, 并在树结构中显示出来




在导入界面添加 Revision 列, 位于浏览按钮列前
confirm 按钮点击后, 将 Revision 信息存储在 .json 文件中, 该字段非必填, 用户可以选择是否填写
在导入界面的每行添加打开图纸文件按钮, 位于浏览按钮后, 点击后可以直接打开对应的图纸PDF文件
生成的 .json 文件的命名方式应该为 "{PO}_import.json", 其中 {PO} 是用户在导入界面输入的 PO 号
